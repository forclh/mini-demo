前端路由模式演示

说明： 本项目演示了使用原生JavaScript实现前端路由的两种模式（Hash模式和History模式）、

运行方式： 为了完整体验所有功能，特别是History模式，请使用Node.js运行服务器：node server.js，然后访问 http://localhost:3000

Hash 模式

使用纯JavaScript实现的Hash模式路由，展示了hashchange事件的使用和路由状态管理的基本原理。

核心技术： window.location.hash、hashchange事件

查看演示

History 模式

使用纯JavaScript实现的History模式路由，展示了HTML5 History API的使用和前端路由的核心原理。

核心技术： history.pushState()、popstate事件

查看演示

两种模式对比

特性 Hash 模式 History 模式
URL 格式 example.com/#/path example.com/path
浏览器支持 所有浏览器 IE10+ 和所有现代浏览器
服务器配置 不需要 需要配置所有路由重定向到 index.html
SEO 友好度 较差（搜索引擎通常忽略 # 后内容） 较好（完整 URL 可被搜索引擎索引）
实现原理 监听 hashchange 事件 使用 History API 和监听 popstate 事件
刷新页面行为 保持在当前路由状态 需要服务器配置，否则可能404
